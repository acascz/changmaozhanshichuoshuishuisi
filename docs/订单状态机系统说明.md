# 订单状态机系统说明

## 架构设计

### 状态流转图
```
待付款 (0)
  ├─> 已付款 (1) ─> 已发货 (2) ─> 已收货 (3) ─> 已完成 (4)
  └─> 已取消 (5)

已付款 (1)
  └─> 退款中 (6) ─> 已退款 (7)
```

### 状态定义
| 状态码 | 状态名称 | 说明 | 是否终态 |
|--------|---------|------|---------|
| 0 | PENDING_PAYMENT | 待付款 | 否 |
| 1 | PAID | 已付款 | 否 |
| 2 | SHIPPED | 已发货 | 否 |
| 3 | DELIVERED | 已收货 | 否 |
| 4 | COMPLETED | 已完成 | ✅ 是 |
| 5 | CANCELLED | 已取消 | ✅ 是 |
| 6 | REFUNDING | 退款中 | 否 |
| 7 | REFUNDED | 已退款 | ✅ 是 |

## 核心组件

### OrderStatus.java
订单状态枚举，提供状态码和描述的映射。

### OrderStateMachineService.java
状态机核心服务，提供：
- 状态转换验证
- 允许的下一个状态查询
- 终态判断

### OrderTimeoutScheduler.java
超时订单自动取消调度器：
- 每 5 分钟检查一次
- 超时时间：30 分钟
- 自动取消并回滚库存

## API 接口

### 支付订单
```
POST /api/order/pay/{id}
响应：{"code": 200, "msg": "支付成功"}
```

### 查询订单状态描述
```
GET /api/order/status/{id}/description
响应：{"code": 200, "data": "待付款"}
```

### 查询允许的下一个状态
```
GET /api/order/status/{id}/allowed-transitions
响应：{"code": 200, "data": ["已付款", "已取消"]}
```

### 检查是否为终态
```
GET /api/order/status/{id}/is-final
响应：{"code": 200, "data": false}
```

## 使用示例

### 1. 更新订单状态（使用状态机验证）
```java
@Autowired
private OrdersService ordersService;

// 支付订单
boolean success = ordersService.payOrder(orderId);

// 取消订单
boolean cancelled = ordersService.cancelOrder(orderId);

// 通用状态更新（会自动验证）
boolean updated = ordersService.updateOrderStatus(orderId, OrderStatus.SHIPPED.getCode());
```

### 2. 查询状态信息
```java
@Autowired
private OrderStateMachineService stateMachineService;

// 检查状态转换是否合法
boolean canPay = stateMachineService.canTransition(
    OrderStatus.PENDING_PAYMENT, 
    OrderStatus.PAID
);

// 获取允许的下一个状态
List<OrderStatus> nextStates = stateMachineService.getAllowedNextStates(
    OrderStatus.PENDING_PAYMENT
);

// 检查是否为终态
boolean isFinal = stateMachineService.isFinalState(OrderStatus.COMPLETED);
```

## 超时取消机制

### 配置
- **检查频率**：每 5 分钟
- **超时时间**：30 分钟
- **触发条件**：订单创建时间 < 当前时间 - 30 分钟

### 流程
```
1. 定时任务触发
   ↓
2. 查询超时的待付款订单
   ↓
3. 验证状态是否允许取消
   ↓
4. 更新订单状态为已取消
   ↓
5. 回滚库存（Redis + 数据库）
   ↓
6. 记录日志
```

### 日志示例
```
[INFO] ========== 开始检查超时订单 ==========
[INFO] 发现 3 个超时订单
[INFO] 超时订单已取消 - 订单：ORD20240403123456
[INFO] 订单 ORD20240403123456 库存已回滚
[INFO] ========== 超时订单检查完成 - 成功：3, 失败：0 ==========
```

## 状态转换规则

### 合法转换
- ✅ PENDING_PAYMENT → PAID
- ✅ PENDING_PAYMENT → CANCELLED
- ✅ PAID → SHIPPED
- ✅ PAID → REFUNDING
- ✅ SHIPPED → DELIVERED
- ✅ DELIVERED → COMPLETED
- ✅ REFUNDING → REFUNDED

### 非法转换
- ❌ PENDING_PAYMENT → SHIPPED（跳过已付款）
- ❌ PAID → PENDING_PAYMENT（回退）
- ❌ COMPLETED → 任何状态（终态）
- ❌ CANCELLED → 任何状态（终态）

## 优势

### 1. 状态安全
- 所有状态转换都经过验证
- 防止非法状态流转
- 自动记录状态变更日志

### 2. 业务清晰
- 状态流转规则集中管理
- 易于理解和维护
- 新状态容易扩展

### 3. 自动化
- 超时订单自动取消
- 库存自动回滚
- 减少人工干预

### 4. 可追溯
- 完整的状态变更日志
- 可查询允许的状态转换
- 便于问题排查

## 注意事项

1. **状态机验证**：
   - 所有状态更新都应通过状态机验证
   - 不要直接调用 `ordersMapper.updateStatus`

2. **超时时间配置**：
   - 当前为 30 分钟
   - 可根据业务需求调整 `OrderTimeoutScheduler.TIMEOUT_MINUTES`

3. **库存回滚**：
   - 取消订单时自动回滚
   - 回滚失败不影响订单取消（记录日志）

4. **终态处理**：
   - 终态订单不可再变更
   - 包括：已完成、已取消、已退款
