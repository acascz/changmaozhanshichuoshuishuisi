### 2.2 客服排队系统

**排队流程**：
```
1. 用户请求客服
2. 加入排队队列
3. 异步线程处理
4. 分配空闲客服
5. 创建客服会话
```

**客服分配算法**：
- 简单轮询：选择第一个空闲客服
- 可扩展：支持负载均衡、优先级等

**并发处理**：
- 线程池大小：2-10
- 队列容量：100
- 空闲超时：60 秒

## 五、数据库设计

### 5.1 chat_group 表结构

```sql
CREATE TABLE `chat_group` (
  `group_id` BIGINT NOT NULL AUTO_INCREMENT,
  `group_no` VARCHAR(32) NOT NULL,          -- 群号（唯一）
  `group_name` VARCHAR(100) NOT NULL,       -- 群名称
  `group_avatar` VARCHAR(255),              -- 群头像
  `group_notice` TEXT,                      -- 群公告
  `creator_id` BIGINT NOT NULL,             -- 创建者 ID
  `max_members` INT DEFAULT 500,            -- 最大成员数
  `member_count` INT DEFAULT 0,             -- 当前成员数
  `status` TINYINT DEFAULT 1,               -- 状态：1-正常，2-禁言，3-解散
  PRIMARY KEY (`group_id`),
  UNIQUE KEY `uk_group_no` (`group_no`)
);
```

### 5.2 chat_session_member 表结构

```sql
CREATE TABLE `chat_session_member` (
  `id` BIGINT NOT NULL AUTO_INCREMENT,
  `session_id` VARCHAR(64) NOT NULL,        -- 会话 ID
  `user_id` BIGINT NOT NULL,                -- 用户 ID
  `role` TINYINT DEFAULT 0,                 -- 角色：0-普通，1-管理员，2-群主
  `session_key` VARCHAR(255),               -- 会话密钥
  `join_time` DATETIME,                     -- 加入时间
  `leave_time` DATETIME,                    -- 退出时间
  UNIQUE KEY `uk_session_user` (`session_id`, `user_id`)
);
```
