# 商家好友系统说明

## 📋 需求说明

### 核心需求
- ✅ **所有用户**（普通用户/商家/管理员）的好友列表里**自动添加一个商家好友**
- ✅ 这个商家好友**删不掉**（固定存在）
- ✅ 用于联系商家客服、商品咨询、售后服务

---

## 🎯 实现方案

### 1. 数据库层面

**创建官方商家客服账号**

**SQL 脚本位置**：[`user_role_migration.sql`](file:///d:/A/changmaozhanshichuoshuishuisi/src/main/resources/sql/user_role_migration.sql)

```sql
-- 创建官方商家客服账号（所有用户的好友）
-- 注意：这个账号是所有用户默认的商家好友，删不掉
INSERT INTO `user` (`username`, `password`, `nickname`, `role`, `create_time`, `update_time`) 
VALUES ('merchant_service', 'MERCHANT_SERVICE_ACCOUNT', '中豆官方客服', 'MERCHANT', NOW(), NOW())
ON DUPLICATE KEY UPDATE `role` = 'MERCHANT';
```

**账号信息**：
- 用户名：`merchant_service`
- 密码：`MERCHANT_SERVICE_ACCOUNT`（固定密码，不可登录）
- 昵称：`中豆官方客服`
- 角色：`MERCHANT`（商家）

### 3. 权限控制

**商家角色验证**

**文件位置**：[`MerchantAuthInterceptor.java`](file:///d:/A/changmaozhanshichuoshuishuisi/src/main/java/com/pdd/mall/config/MerchantAuthInterceptor.java)

```java
String userRole = currentUser.getRole();
if (userRole == null || (!"MERCHANT".equals(userRole) && !"ADMIN".equals(userRole))) {
    response.setStatus(403);
    response.getWriter().write("权限不足：只有商家才能访问");
    return false;
}
```

**权限说明**：
- ✅ 商家客服账号角色为 `MERCHANT`
- ✅ 可以访问商家后台功能
- ✅ 普通用户无法删除此好友（前端固定显示）

## 🔧 使用场景

### 场景 1：用户咨询商品

1. **用户打开聊天页面**
   ```
   访问：http://localhost:8080/index.html
   ```

2. **点击"中豆商家客服"**
   - 进入聊天详情页
   - 可以发送商品咨询消息

3. **商家客服回复**
   - 商家在后台收到消息
   - 通过商家后台回复用户

---

### 场景 2：用户申请售后

1. **用户点击商家客服**
   - 发送售后申请

2. **商家客服处理**
   - 引导用户到退款页面
   - 或直接处理售后

---

### 场景 3：商家登录

1. **商家访问发货管理**
   ```
   访问：http://localhost:8080/delivery-management.html
   ```

2. **权限验证通过**
   - 角色为 `MERCHANT`
   - 可以正常访问

3. **普通用户尝试访问**
   - 角色为 `USER`
   - 被拦截器拒绝（403）

## 🔒 安全性说明

### 1. 商家客服账号保护

**不可删除**：
- ✅ 前端固定显示（HTML 硬编码）
- ✅ 数据库账号存在
- ✅ 用户无法通过前端操作删除

**不可登录**：
- ✅ 固定密码（`MERCHANT_SERVICE_ACCOUNT`）
- ✅ 建议在生产环境禁用此账号的登录功能

---

### 2. 角色隔离

**普通用户（USER）**：
- ✅ 可以看到商家客服好友
- ✅ 可以联系商家客服
- ❌ 无法删除商家客服好友
- ❌ 无法访问商家后台

**商家（MERCHANT）**：
- ✅ 可以看到商家客服好友
- ✅ 可以访问商家后台
- ✅ 可以处理用户咨询

**管理员（ADMIN）**：
- ✅ 所有权限
- ✅ 可以管理商家账号
