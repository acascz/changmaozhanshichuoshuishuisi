# 库存管理系统说明

## 架构设计

### 核心机制
1. **Redis 预扣库存** - 高性能，防止超卖
2. **异步同步数据库** - MQ 削峰填谷（当前为同步确认，可后续改为 MQ）
3. **定期对账机制** - 保证 Redis 和数据库库存一致性

## 工作流程

### 下单流程
```
1. 用户下单
   ↓
2. Redis SETNX 获取库存锁（防止并发）
   ↓
3. 检查并扣减 Redis 库存
   ↓
4. 记录预扣日志（用于对账）
   ↓
5. 创建订单
   ↓
6. 释放锁（延迟释放）
```

### 支付成功流程
```
1. 支付回调成功
   ↓
2. 确认库存扣减
   ↓
3. 更新数据库库存
   ↓
4. 标记对账记录为已确认
```

### 取消订单流程
```
1. 用户取消订单
   ↓
2. 回滚 Redis 库存
   ↓
3. 记录回滚日志
   ↓
4. 更新订单状态
```

## API 接口

### 查询库存
```
GET /api/order/stock/{productId}
响应：{"code": 200, "data": 100}
```

### 手动对账（管理员）
```
POST /api/order/reconciliation
响应：{"code": 200, "msg": "对账完成"}
```

## 定时任务

### 每小时对账
- 时间：每小时整点
- 作用：检查并修复 Redis 和数据库库存不一致

### 每日对账
- 时间：凌晨 2 点
- 作用：全面对账，生成对账报告

## 关键代码位置

- 库存服务：`InventoryService.java`
- 订单服务：`OrdersService.java`
- 对账调度：`InventoryReconciliationScheduler.java`
- 缓存初始化：`InventoryCacheInitializer.java`

## 注意事项

1. **系统启动时**会自动从数据库同步库存到 Redis
2. **库存不足时**会立即返回失败，不会创建订单
3. **订单取消时**会自动回滚库存
4. **对账机制**会以数据库为准修复 Redis 库存

## 性能优化

1. **读操作**：优先从 Redis 读取库存，未命中才查数据库
2. **写操作**：先扣减 Redis，异步同步到数据库
3. **并发控制**：使用 Redis SETNX 保证原子性
4. **锁超时**：5 分钟自动释放，防止死锁

- 库存预扣成功率
- 对账发现的不一致数量
