## 安全性保护

### 1. 防止重复提交

使用自定义注解和 Redis 实现：

```java
@PreventDuplicate(interval = 1000, message = "操作过于频繁")
@PostMapping("/send")
public Result sendMessage(...) {
    // 业务逻辑
}
```

**实现原理**：
- 使用 Redis 记录请求 key
- 在指定时间间隔内拒绝重复请求
- 返回 HTTP 429 状态码

### 2. 限流保护

使用令牌桶算法实现 API 限流：

```java
@Autowired
private RateLimiter rateLimiter;

public void businessMethod() {
    if (!rateLimiter.isAllowed("api_key", 10, 60)) {
        throw new RuntimeException("请求过于频繁");
    }
    // 业务逻辑
}
```

### 3. CORS 跨域配置

```java
@Configuration
public class WebSecurityConfig implements WebMvcConfigurer {
    @Override
    public void addCorsMappings(CorsRegistry registry) {
        registry.addMapping("/**")
                .allowedOrigins("http://localhost:8080")
                .allowedMethods("GET", "POST", "PUT", "DELETE")
                .allowCredentials(true);
    }
}
```

### 4. 会话管理

- 使用 Session 存储用户登录状态
- 拦截器验证用户身份
- 敏感操作需要重新验证

## 数据加密

### 1. 密码加密

使用 MD5 + Salt 加密：

```java
String encryptedPassword = DigestUtils.md5DigestAsHex(password.getBytes());
```

### 2. 敏感数据加密

使用 AES 加密存储：

```java
// 加密
String encrypted = EncryptionUtil.aesEncrypt(phone, secretKey);

// 解密
String decrypted = EncryptionUtil.aesDecrypt(encrypted, secretKey);
```

### 3. 数据脱敏

数据库视图自动脱敏：

```sql
CREATE VIEW v_user_safe AS
SELECT 
    id,
    username,
    nickname,
    CONCAT(LEFT(phone, 3), '****', RIGHT(phone, 4)) AS phone_masked
FROM user;
```

## 并发控制

### 1. 数据库事务

使用 `@Transactional` 保证数据一致性：

```java
@Transactional(rollbackFor = Exception.class)
public void approveFriendRequest(Long requestId, Long userId) {
    // 1. 更新申请状态
    // 2. 添加好友关系
    // 要么全部成功，要么全部回滚
}
```

### 2. Redis 分布式锁

```java
String lockKey = "lock:friend_request:" + requestId;
Boolean locked = redisTemplate.opsForValue()
    .setIfAbsent(lockKey, "1", 10, TimeUnit.SECONDS);

if (Boolean.TRUE.equals(locked)) {
    try {
        // 执行业务逻辑
    } finally {
        redisTemplate.delete(lockKey);
    }
}
```

### 3. 乐观锁

使用版本号控制：

```java
@Version
private Integer version;

// 更新时检查版本号
UPDATE friend_request 
SET status = 1, version = version + 1 
WHERE id = #{id} AND version = #{version}
```

## 常见问题

### Q1: 好友申请发送失败？

**检查项**：
1. 数据库表 `friend_request` 是否创建
2. 是否已经是好友关系
3. 是否已有待处理的申请

### Q2: 搜索用户返回空？

**检查项**：
1. 关键词是否匹配用户名、昵称或 IP
2. UserMapper.xml 中是否添加了 `searchByKeyword` 方法
3. 用户表是否有 `ip` 字段

### Q3: 防止重复提交不生效？

**检查项**：
1. Redis 服务是否正常运行
2. 拦截器是否正确配置
3. 方法上是否添加了 `@PreventDuplicate` 注解
