## 配置步骤

### 步骤 1：数据库配置

```sql
-- 确保发货单表存在
CREATE TABLE IF NOT EXISTS `delivery_order` (
    `id` BIGINT PRIMARY KEY AUTO_INCREMENT,
    `order_id` BIGINT NOT NULL,
    `user_id` BIGINT NOT NULL,
    `logistics_company` VARCHAR(50),
    `logistics_no` VARCHAR(50),
    `status` INT DEFAULT 0,
    `delivery_time` DATETIME,
    `create_time` DATETIME,
    `update_time` DATETIME,
    INDEX `idx_order_id` (`order_id`),
    INDEX `idx_logistics_no` (`logistics_no`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 确保物流轨迹表存在
CREATE TABLE IF NOT EXISTS `logistics_track` (
    `id` BIGINT PRIMARY KEY AUTO_INCREMENT,
    `logistics_no` VARCHAR(50),
    `logistics_company` VARCHAR(50),
    `order_id` BIGINT,
    `user_id` BIGINT,
    `track_info` TEXT,
    `status` INT,
    `create_time` DATETIME,
    `update_time` DATETIME,
    INDEX `idx_logistics_no` (`logistics_no`),
    INDEX `idx_order_id` (`order_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

## 使用流程

### 商家端操作流程

1. **登录商家后台**
   ```
   访问：http://localhost:8080/delivery-management.html
   ```

2. **查看待发货订单**
   - 页面自动加载所有订单

3. **填写物流信息**
   - 点击"发货"按钮

## 常见问题

### Q1: 商家权限如何验证？

**A**: 目前通过 `MerchantAuthInterceptor` 验证登录状态，等用户系统集成角色后，会添加角色验证。

---

### Q2: 物流 API 必须配置吗？

**A**: 不是必须的。如果不配置物流 API：
- ✅ 商家仍可手动发货
- ❌ 无法自动拉取物流轨迹
- ✅ 可以手动更新物流信息

---

### Q3: WebSocket 推送失败怎么办？

**A**: WebSocket 推送失败不影响发货流程：
- ✅ 发货操作正常完成
- ⚠️ 用户暂时收不到推送
- ✅ 用户刷新页面仍能看到最新状态

**解决方案**：
1. 检查 WebSocket 连接状态
2. 检查用户是否在线
3. 使用轮询作为备用方案
