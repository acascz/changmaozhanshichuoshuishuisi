## 📝 配置文件位置

### 后端配置

**文件路径**：
```
src/main/resources/application-prod.yml
```

**快速配置**：
```bash
# 1. 复制模板
cp src/main/resources/application-logistics-template.yml src/main/resources/application-prod.yml

# 2. 编辑配置
vim src/main/resources/application-prod.yml
```

**必须配置的密钥**（共 11 个）：

```yaml
# 高德地图（3 个）
amap:
  key: 你的 Web 服务 API Key
  securityCode: 你的安全密钥
  webKey: 你的 Web 端 API Key

# 快递 100（2 个）
kuaidi100:
  customer: 你的客户 ID
  key: 你的 API Key

# 阿里云短信（4 个）
aliyun:
  sms:
    accessKeyId: 你的 Access Key ID
    accessKeySecret: 你的 Access Key Secret
    signName: 你的短信签名
    templateCode:
      delivery: SMS_发货通知模板 CODE
      sign: SMS_签收通知模板 CODE
      exception: SMS_异常通知模板 CODE

# 雪花算法（1 个，集群部署时必须）
snowflake:
  machineId: 0  # 每台机器必须不同（0-1023）
```

### 前端配置

**文件路径**：

## 🔑 密钥申请详细步骤

### 1. 高德地图 API Key（3 个）

**步骤**：
1. 访问 https://lbs.amap.com/
2. 注册/登录账号
3. 进入「控制台」→「应用管理」→「我的应用」
4. 点击「创建新应用」
5. 创建 2 个 Key：

**Key 1: Web 服务 API Key**
- Key 名称：物流系统后端
- 服务平台：Web 服务
- 白名单：留空（生产环境填写服务器 IP）

**Key 2: Web 端 API Key**
- Key 名称：物流系统前端
- 服务平台：Web 端（JS API）
- 白名单：localhost（开发环境）或 your-domain.com（生产环境）

6. 安全密钥在「安全密钥管理」中查看

**预计耗时**：15 分钟

---

### 2. 快递 100 API Key（2 个）

**步骤**：
1. 访问 https://api.kuaidi100.com/
2. 注册/登录账号
3. 实名认证（需要营业执照或身份证）
4. 进入「个人中心」→「接口配置」
5. 获取 Customer 和 Key

**套餐选择**：
- 免费版：100 次/天（测试用）
- 电子面单版：¥600/年（10 万次查询）
- 快递推送版：¥1200/年（20 万次查询，推荐）

**预计耗时**：30 分钟（含实名认证）

---

### 3. 阿里云短信（4 个配置项）

**步骤**：
1. 访问 https://www.aliyun.com/product/sms
2. 注册/登录账号
3. 实名认证（企业或个人）
4. 开通短信服务
5. 进入「短信控制台」

**5.1 创建 Access Key**
- 访问 https://ram.console.aliyun.com/manage/ak
- 创建 Access Key
- 记录 AccessKeyId 和 AccessKeySecret

**5.2 添加签名**
- 进入「签名管理」
- 点击「添加签名」
- 签名内容：公司名称或网站名称
- 审核时间：1-2 小时

**5.3 添加模板**
- 进入「模板管理」
- 点击「添加模板」
- 模板类型：验证码/通知/推广
- 模板内容示例：
  ```
  您的订单${orderNo}已发货，请注意查收。
  您的订单${orderNo}已签收，感谢您的支持。
  您的订单${orderNo}出现异常，请及时联系客服。
  ```
- 审核时间：1-2 小时

**预计耗时**：2 小时（含审核）

## ⚠️ 常见问题

### Q1: 地图服务调用失败？
**A**: 检查以下几点：
1. API Key 是否正确（区分 Web 服务和 Web 端）
2. 安全密钥是否配置
3. 域名白名单是否添加
4. 调用次数是否超限

### Q2: 物流轨迹查询失败？
**A**: 检查以下几点：
1. 快递 100 API Key 是否有效
2. 物流单号格式是否正确
3. 物流公司编码是否正确
4. 账户余额是否充足

### Q3: 短信发送失败？
**A**: 检查以下几点：
1. Access Key 是否正确
2. 短信签名是否审核通过
3. 短信模板是否审核通过
4. 手机号格式是否正确
5. 账户余额是否充足

### Q4: 分布式 ID 重复？
**A**: 检查以下几点：
1. 每台机器的 machineId 是否唯一
2. 系统时间是否同步（使用 NTP）
3. 是否有时钟回拨问题
