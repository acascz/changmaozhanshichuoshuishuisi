# 第三方登录配置指南

## 📋 概述

本系统支持 QQ、微信、邮箱三种第三方登录方式，所有配置都预留了密钥位置，**验证机制必须保留，不能跳过**。

---

## 🔐 配置方式

### 方式一：application.yml 配置（开发环境）

在 `application.yml` 中找到 `third-party.login` 配置项，填入真实密钥：

```yaml
third-party:
  login:
    qq:
      app-id: "你的 QQ AppID"
      app-key: "你的 QQ AppKey"
      redirect-uri: "http://localhost:8080/api/auth/qq/callback"
    
    wechat:
      app-id: "你的微信 AppID"
      app-secret: "你的微信 AppSecret"
      redirect-uri: "http://localhost:8080/api/auth/wechat/callback"
    
    email:
      smtp-host: "smtp.qq.com"
      smtp-port: 465
      from-email: "your-email@qq.com"
      password: "SMTP 授权码"
      ssl-enabled: true
```

### 方式二：环境变量配置（生产环境 ⭐推荐）

**不要将真实密钥提交到代码仓库！** 使用环境变量：

```bash
# Linux/Mac
export THIRD_PARTY_LOGIN_QQ_APP_ID="your_qq_appid"
export THIRD_PARTY_LOGIN_QQ_APP_KEY="your_qq_appkey"
export THIRD_PARTY_LOGIN_WECHAT_APP_ID="your_wechat_appid"
export THIRD_PARTY_LOGIN_WECHAT_APP_SECRET="your_wechat_secret"
export THIRD_PARTY_LOGIN_EMAIL_SMTP_HOST="smtp.qq.com"
export THIRD_PARTY_LOGIN_EMAIL_FROM_EMAIL="noreply@example.com"
export THIRD_PARTY_LOGIN_EMAIL_PASSWORD="your_smtp_auth_code"
```

```powershell
# Windows PowerShell
$env:THIRD_PARTY_LOGIN_QQ_APP_ID="your_qq_appid"
$env:THIRD_PARTY_LOGIN_QQ_APP_KEY="your_qq_appkey"
# ... 其他环境变量
```

### 方式三：配置中心（生产环境 ⭐推荐）

使用 Nacos、Apollo 等配置中心管理密钥：

```yaml
# Nacos 配置
third-party:
  login:
    qq:
      app-id: ${QQ_APP_ID:****}
      app-key: ${QQ_APP_KEY:****}
    wechat:
      app-id: ${WECHAT_APP_ID:****}
      app-secret: ${WECHAT_APP_SECRET:****}
    email:
      password: ${EMAIL_SMTP_PASSWORD:****}
```

## 📱 QQ 登录配置

### 1. 申请 QQ 互联

1. 访问：https://connect.qq.com/
2. 登录 QQ 账号
3. 创建网站应用
4. 通过审核获取 AppID 和 AppKey

### 2. 配置回调域

在 QQ 互联后台配置回调域：
```
http://localhost:8080/api/auth/qq/callback  (开发环境)
https://yourdomain.com/api/auth/qq/callback  (生产环境)
```

### 3. 验证机制

**必须保留的验证：**
- ✅ 验证授权码（code）的有效性
- ✅ 验证 state 参数防止 CSRF 攻击
- ✅ 验证用户 OpenID 是否已绑定账号
- ✅ 强制绑定手机号（安全要求）

## 📧 邮箱登录配置

### 1. 开启 SMTP 服务

#### QQ 邮箱
1. 登录 QQ 邮箱
2. 设置 → 账户
3. 开启 POP3/SMTP 服务
4. 生成授权码（不是邮箱密码）

#### 163 邮箱
1. 登录 163 邮箱
2. 设置 → POP3/SMTP/IMAP
3. 开启 SMTP 服务
4. 设置授权码

### 2. 配置 SMTP 参数

```yaml
email:
  smtp-host: "smtp.qq.com"    # QQ 邮箱 SMTP 服务器
  smtp-port: 465              # SSL 端口
  from-email: "your-email@qq.com"
  password: "SMTP 授权码"      # ⭐ 不是邮箱登录密码
  ssl-enabled: true
```

### 3. 验证机制

**必须保留的验证：**
- ✅ 验证邮箱格式
- ✅ 发送频率限制（60 秒/次）
- ✅ 验证码有效期（5 分钟）
- ✅ 验证码一次性使用
- ✅ 强制绑定手机号（安全要求）

## 🔒 安全要求

### ⭐ 密钥管理

1. **生产环境必须使用环境变量或配置中心**
2. **不要将真实密钥提交到 Git**
3. **定期轮换密钥**
4. **限制密钥访问权限**

### ⭐ 验证机制

**以下验证机制不能跳过：**

| 验证项 | QQ | 微信 | 邮箱 | 说明 |
|--------|-----|------|------|------|
| 授权码验证 | ✅ | ✅ | ✅ | 验证 code 有效性 |
| CSRF 防护 | ✅ | ✅ | ✅ | 验证 state 参数 |
| 用户绑定验证 | ✅ | ✅ | ✅ | 检查是否已绑定 |
| 手机号绑定 | ✅ | ✅ | ✅ | 强制绑定手机号 |
| 频率限制 | ✅ | ✅ | ✅ | 防止暴力破解 |
| Token 验证 | ✅ | ✅ | ❌ | 验证 Access Token |

### ⭐ 日志记录

所有第三方登录操作必须记录日志：
```java
log.info("QQ 登录 - OpenID: {}, 用户：{}", openId, nickname);
log.warn("QQ 登录失败 - 原因：{}", errorMessage);
```

## 🚨 常见问题

### Q1: 提示"API 尚未接入"
**A:** 检查是否配置了密钥，如果没有配置，系统会抛出异常提示。

### Q2: 扫码后无法登录
**A:** 检查回调域配置是否正确，确保与后台配置一致。

### Q3: 邮箱验证码发送失败
**A:** 检查 SMTP 配置，确认使用的是授权码而不是邮箱密码。

### Q4: 是否需要修改验证机制？
**A:** **绝对不可以！** 所有验证机制都是强制要求，不能跳过或修改。
