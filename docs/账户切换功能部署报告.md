# 账户切换功能 - 完整部署报告

## 📋 项目概述

基于我们项目的实际架构，已完成账户切换功能的完整实现和部署配置。本报告详细记录了所有部署步骤、配置文件和验证方法。

## 🎯 功能特性

### 核心功能
- ✅ **账户绑定** - 最多绑定3个账户，需验证码验证
- ✅ **账户切换** - 点击卡片直接切换，无需重复验证
- ✅ **账户解绑** - 二次确认解绑操作
- ✅ **安全风控** - IP+设备双维度限流和监控

### 安全特性
- ✅ 全链路数据加密传输和存储
- ✅ 分布式锁防止并发冲突
- ✅ 请求签名防伪造攻击
- ✅ 幂等性检查防重复操作

## 🗄️ 数据库部署

### 执行文件
- **位置**: `database/account_switch_complete.sql`
- **执行命令**: 
  ```sql
  mysql -u root -p mung_bean_cake_mall < database/account_switch_complete.sql
  ```

### 创建的表结构

| 表名 | 功能说明 | 关键特性 |
|------|----------|----------|
| `user_session` | 用户会话管理 | 加密存储、设备绑定 |
| `account_relationship` | 账户关联关系 | 最多3个绑定限制 |
| `verification_record` | 验证码记录 | 5分钟过期机制 |
| `operation_log` | 操作审计日志 | 完整操作追溯 |
| `security_control` | 安全风控 | IP/设备黑名单 |
| `rate_limit_record` | 限流记录 | 请求频率控制 |

### 安全特性
- 所有敏感字段AES加密存储
- 复合索引优化查询性能
- 定时任务自动清理过期数据
- 视图简化复杂查询

## 🔧 Redis配置

### 配置文件
- **位置**: `config/account-switch-config.yml`
- **架构**: 基于现有单机Redis

### 缓存策略
| 缓存类型 | 过期时间 | 用途 |
|----------|----------|------|
| 会话缓存 | 7天 | 用户会话信息 |
| 验证码缓存 | 5分钟 | 验证码验证 |
| 限流缓存 | 1分钟 | 请求频率控制 |
| 黑名单缓存 | 30分钟 | 恶意行为封禁 |

### 环境配置
- **开发环境**: 简化配置，支持调试
- **生产环境**: 严格安全配置

## 🚀 应用部署

### 配置文件
- **位置**: `src/main/resources/application-account-switch.yml`
- **集成方式**: 与现有Spring Boot应用无缝集成

### API接口

| 方法 | 路径 | 功能 | 请求头 |
|------|------|------|--------|
| GET | `/api/account-switch/bound-accounts` | 获取绑定账户列表 | Authorization, Device-Id |
| POST | `/api/account-switch/bind` | 绑定账户 | Authorization, Device-Id |
| POST | `/api/account-switch/switch` | 切换账户 | Authorization, Device-Id |
| POST | `/api/account-switch/unbind` | 解绑账户 | Authorization, Device-Id |

### 定时任务
- **清理过期会话**: 每天凌晨2点执行
- **清理过期验证码**: 每5分钟执行一次

#### 4. 业务监控
- 绑定数量统计和成功率
- 切换成功率监控
- 验证码发送和验证统计

### 告警机制

#### 告警级别
- **严重** - 系统级故障
- **警告** - 业务异常
- **信息** - 运行状态

#### 告警渠道
- **开发环境**: 控制台输出
- **生产环境**: 邮件 + SMS告警

#### 告警规则
- 切换成功率低于80%（持续5分钟）

## 🚀 部署步骤

### 第一步：数据库部署

```bash
# 1. 登录MySQL
mysql -u root -p

# 2. 选择数据库
USE mung_bean_cake_mall;

# 3. 执行SQL脚本
source database/account_switch_complete.sql;
```

### 第二步：应用部署

#### 开发环境
```bash
# 1. 启动应用
mvn spring-boot:run

# 2. 验证服务
curl http://localhost:8080/api/account-switch/bound-accounts
```

#### 生产环境
```bash
# 1. 打包应用
mvn clean package

# 2. 部署运行
java -jar target/changmaozhanshichuoshuishuisi.jar \
  --spring.profiles.active=prod
```

### 第三步：前端集成

#### 1. 引入资源文件
在HTML文件中添加：
```html
<!-- CSS样式 -->
<link rel="stylesheet" href="/css/account-switch.css">

<!-- JavaScript逻辑 -->
<script src="/js/account-switch.js"></script>
```

#### 2. 添加切换按钮
在个人中心页面添加：
```html
<button class="switch-account-btn" onclick="showAccountCardsModal()">
  切换账户
</button>
```

#### 3. 添加模态框结构
在页面底部添加模态框HTML结构。

### 第四步：监控配置

#### 1. 配置监控参数
根据环境修改 `config/monitoring-config.yml`

#### 2. 设置告警接收人
配置生产环境的邮件和SMS接收人

#### 3. 验证监控功能
```bash
# 检查健康状态
curl http://localhost:8080/actuator/health

# 查看指标
curl http://localhost:8080/actuator/metrics
```
