# 支付系统安全架构说明

## 核心安全机制

### 1. 支付签名验证（防伪造）
**目的**：防止恶意伪造支付回调请求

**机制**：
- 使用 HMAC-SHA256 签名算法
- 所有支付回调请求必须携带签名
- 签名包含：订单号、金额、时间戳、随机数

**签名生成流程**：
```
1. 收集参数：orderNo, amount, userId
2. 生成时间戳和随机数
3. 参数按字母顺序排序
4. 拼接：SECRET_KEY + "&" + 参数字符串
5. HMAC-SHA256 计算签名
6. Base64 编码返回
```

**验证流程**：
```
1. 提取请求中的签名
2. 使用相同算法重新计算签名
3. 比较两个签名是否一致
4. 验证时间戳有效性（防重放攻击）
```

### 2. 支付回调幂等处理（防重复）
**目的**：防止支付平台重试导致重复处理

**机制**：
- Redis SETNX 分布式锁
- 订单号作为锁的 Key
- 交易号作为锁的 Value
- 24 小时过期时间

**幂等逻辑**：
```java
// 尝试获取锁
Boolean locked = redisTemplate.opsForValue()
    .setIfAbsent(callbackKey, transactionId, 24, TimeUnit.HOURS);

if (!locked) {
    // 已处理过，检查是否是同一个交易号
    String existingTransactionId = redisTemplate.opsForValue().get(callbackKey);
    if (transactionId.equals(existingTransactionId)) {
        return true;  // 同一交易号，返回成功
    } else {
        return false; // 不同交易号，拒绝
    }
}
```

**处理流程**：
```
1. 收到支付回调
   ↓
2. Redis SETNX 尝试加锁
   ↓
3. 加锁失败 → 检查交易号是否一致
   ├─ 一致 → 返回成功（幂等）
   └─ 不一致 → 拒绝（异常）
   ↓
4. 加锁成功 → 处理支付
   ├─ 验证签名
   ├─ 校验金额
   ├─ 检查订单状态
   ├─ 更新订单状态
   └─ 记录支付信息
   ↓
5. 异常处理 → 删除锁，允许重试
```

### 3. 支付金额校验（防篡改）
**目的**：防止篡改支付金额

**机制**：
- 查询数据库订单金额
- 与回调金额精确比较（BigDecimal）
- 不一致则拒绝

**代码示例**：
```java
Orders order = ordersMapper.findByOrderNo(orderNo);
if (order.getTotalAmount().compareTo(amount) != 0) {
    log.error("支付金额不匹配 - 订单：{}, 订单金额：{}, 支付金额：{}", 
            orderNo, order.getTotalAmount(), amount);
    return false;
}
```

### 4. 订单状态验证（状态机）
**目的**：防止重复支付、非法状态转换

**机制**：
- 使用订单状态机验证
- 只允许待付款订单支付
- 已支付订单拒绝再次支付

**代码示例**：
```java
if (!stateMachineService.canTransition(order.getStatus(), OrderStatus.PAID.getCode())) {
    log.error("订单状态不允许支付 - 订单：{}, 当前状态：{}", orderNo, order.getStatus());
    return false;
}
```

## 使用示例

### 前端调用支付流程
```javascript
// 1. 请求后端生成签名
const signResponse = await fetch('/api/payment/generate-sign', {
    method: 'POST',
    body: JSON.stringify({
        orderNo: 'ORD20240403123456',
        amount: 99.99,
        userId: 123
    })
});
const { data } = await signResponse.json();

// 2. 调用支付平台（携带签名）
await payPlatform.pay({
    orderNo: data.orderNo,
    amount: data.amount,
    timestamp: data.timestamp,
    nonce: data.nonce,
    sign: data.sign
});

// 3. 支付平台回调后端
// 后端自动验证签名、幂等处理
```

## 注意事项

### 1. 密钥管理
⚠️ 签名密钥必须保密  
⚠️ 定期更换密钥  
⚠️ 不同环境使用不同密钥  

### 2. 时间戳验证
⚠️ 建议验证时间戳在 5 分钟内  
⚠️ 防止重放攻击  

### 3. 幂等窗口
⚠️ 当前设置为 24 小时  
⚠️ 可根据业务调整  

### 4. 异常处理
⚠️ 异常时自动删除锁，允许重试  
⚠️ 记录详细日志便于排查  

### 5. 测试接口
⚠️ 测试接口仅限开发环境使用  
⚠️ 生产环境必须禁用  

## 扩展建议

### 1. 异步通知
- 支付成功后发送 MQ 消息
- 通知其他系统（库存、物流、积分）

### 2. 对账机制
- 每日与支付平台对账
- 发现异常订单及时处理

### 3. 退款支持
- 实现退款接口
- 退款同样需要签名验证和幂等处理

### 4. 多渠道支付
- 支持微信、支付宝、银联
- 不同渠道使用不同签名密钥
