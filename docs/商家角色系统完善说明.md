```javascript
async function checkMerchantAuth() {
    const response = await fetch('/api/user/check-merchant', {
        credentials: 'include'
    });
    
    const result = await response.json();
    
    if (result.code !== 200 || !result.data) {
        alert('权限不足：只有商家才能访问此页面');
        window.location.href = '/login.html';
        return false;
    }
}
```

## 数据库迁移

### 步骤 1：执行迁移脚本

**文件位置**：[`user_role_migration.sql`](file:///d:/A/changmaozhanshichuoshuishuisi/src/main/resources/sql/user_role_migration.sql)

```sql
-- 1. 添加 role 字段
ALTER TABLE `user` 
ADD COLUMN `role` VARCHAR(20) DEFAULT 'USER' 
COMMENT '用户角色：USER-普通用户，MERCHANT-商家，ADMIN-管理员' 
AFTER `phone`;

-- 2. 设置默认角色
UPDATE `user` SET `role` = 'USER' WHERE `role` IS NULL;

-- 3. 创建测试商家账号
INSERT INTO `user` (`username`, `password`, `nickname`, `role`, `create_time`, `update_time`) 
VALUES ('merchant_test', 'merchant123', '测试商家', 'MERCHANT', NOW(), NOW())
ON DUPLICATE KEY UPDATE `role` = 'MERCHANT';

-- 4. 创建索引
CREATE INDEX `idx_user_role` ON `user` (`role`);
```

**执行方式**：
```bash
mysql -u root -p mung_bean_cake_mall < src/main/resources/sql/user_role_migration.sql
```

## 权限验证流程

### 完整流程

```
用户访问 delivery-management.html
         ↓
1. 前端调用 /api/user/check-merchant
         ↓
2. 后端从 session 获取 currentUser
         ↓
3. 验证 role 字段
         ↓
   ┌────┴────┐
   │         │
 MERCHANT  USER
   │         │
   ↓         ↓
 允许访问   拒绝访问 (403)
```

## 常见问题

### Q1: 现有用户如何处理？

**A**: 迁移脚本会自动为所有现有用户设置 `role = 'USER'`

---

### Q2: 如何创建第一个商家账号？

**A**: 有三种方式：
1. 执行迁移脚本（创建测试账号 `merchant_test`）
2. 访问注册页面注册
3. 直接在数据库插入：
   ```sql
   INSERT INTO user (username, password, role) 
   VALUES ('my_store', 'password', 'MERCHANT');
   ```

---

### Q3: 如何升级普通用户为商家？

**A**: 直接更新数据库：
```sql
UPDATE user SET role = 'MERCHANT' WHERE username = '用户名';
```

---

### Q4: 管理员如何访问商家功能？

**A**: 管理员（`role = 'ADMIN'`）可以访问所有商家功能，无需额外配置。
