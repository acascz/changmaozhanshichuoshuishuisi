# 数据加密系统说明

## 架构设计

### 1. 传输加密
- **AES-GCM 加密**：防止数据窃听
- **HMAC-SHA256 签名**：防止数据篡改
- **时间戳 + 随机数**：防止重放攻击

### 2. 存储加密
- **敏感数据加密**：手机号、身份证、银行卡、地址等
- **透明加解密**：业务层无感知
- **容错机制**：解密失败返回原始数据（兼容旧数据）

## 核心组件

### AESUtil.java
- AES-GCM 256 位加密
- 随机 IV 保证每次加密结果不同
- 支持密钥轮换

### SignUtil.java
- HMAC-SHA256 签名
- 参数自动排序
- 签名有效期 5 分钟

### EncryptionService.java
- 统一加密服务接口
- 支持多种敏感数据类型
- 自动脱敏日志

### SignatureInterceptor.java
- 请求签名验证拦截器
- 可配置的路径过滤
- 仅对写操作进行验证

## 使用示例

### 1. 加密存储敏感数据
```java
@Autowired
private EncryptionService encryptionService;

// 加密手机号
String encryptedPhone = encryptionService.encryptPhone("13800138000");

// 解密手机号
String phone = encryptionService.decryptPhone(encryptedPhone);
```

### 2. 生成请求签名
```java
@Autowired
private SignUtil signUtil;

Map<String, Object> params = new HashMap<>();
params.put("userId", "123");
params.put("action", "create_order");

long timestamp = signUtil.generateTimestamp();
String nonce = signUtil.generateNonce();
String sign = signUtil.generateSign(params, timestamp, nonce);

// 将 timestamp、nonce、sign 添加到请求参数中
```

## 密钥管理

### 当前实现
- AES 密钥：硬编码在 `AESUtil.java`
- 签名密钥：硬编码在 `SignUtil.java`

### 生产环境建议
1. 从配置中心读取密钥（如 Apollo、Nacos）
2. 使用密钥管理系统（KMS）
3. 定期轮换密钥
4. 密钥分片存储

## 加密算法说明

### AES-GCM
- **模式**：Galois/Counter Mode
- **密钥长度**：256 位
- **IV 长度**：96 位（12 字节）
- **认证标签**：128 位
- **优势**：认证加密，防篡改

### HMAC-SHA256
- **算法**：Hash-based Message Authentication Code
- **哈希函数**：SHA-256
- **优势**：计算快速，安全性高

## 性能影响

### AES 加密
- 加密速度：~100MB/s
- 解密速度：~100MB/s
- 单次加密耗时：< 1ms

### 签名验证
- 生成签名：< 0.5ms
- 验证签名：< 1ms

## 注意事项

1. **密钥安全**：
   - 不要将密钥提交到代码仓库
   - 使用环境变量或配置中心管理

2. **性能优化**：
   - 批量数据加密考虑使用硬件加速
   - 签名验证可缓存结果（短时间内）

3. **兼容性**：
   - 旧数据解密失败时返回原始数据
   - 签名验证默认跳过（逐步启用）

4. **日志脱敏**：
   - 加密服务自动脱敏日志
   - 显示前 2 位和后 2 位
