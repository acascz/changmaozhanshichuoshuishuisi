# 账户切换功能机制报告

## 📋 报告概述

本报告详细记录了基于项目现有架构的账户切换功能的完整实现机制。该系统实现了安全、高并发的账户绑定、切换和解绑功能，最多支持绑定3个账户。

## 🎯 功能特性

### 核心功能
- **账户绑定** - 最多绑定3个账户，需验证码验证
- **账户切换** - 点击卡片直接切换，无需重复验证
- **账户解绑** - 二次确认解绑操作
- **会话管理** - 安全的会话创建和更新机制

### 安全特性
- **IP+设备双维度限流** - 防止恶意请求
- **分布式锁** - 防止并发冲突
- **幂等检查** - 防止重复操作
- **黑名单机制** - 自动封禁恶意行为
- **数据脱敏** - 敏感信息保护

### 性能特性
- **Redis缓存** - 高性能会话和验证码存储
- **Lua脚本原子操作** - 确保数据一致性
- **异步处理** - 非阻塞操作提升响应速度
- **熔断器机制** - 服务降级保护

### 1. 账户绑定机制

#### 绑定流程
```
用户请求绑定 → 验证会话 → 发送验证码 → 验证验证码 → 检查绑定限制 → 创建绑定关系
```

#### 关键实现
```java
// 绑定账户核心逻辑
public Result bindAccount(String token, String deviceId, String targetPhone, 
                         String verificationCode) {
    // 1. 会话验证
    Long currentUserId = validateSession(token, deviceId);
    
    // 2. 限流检查
    if (!checkRateLimit(currentUserId, "bind")) {
        return Result.error("操作过于频繁");
    }
    
    // 3. 验证码验证
    if (!verifyCode(targetPhone, verificationCode, "bind_account")) {
        return Result.error("验证码错误");
    }
    
    // 4. 绑定限制检查
    if (isBindLimitReached(currentUserId)) {
        return Result.error("绑定数量已达上限");
    }
    
    // 5. 创建绑定关系
    createAccountRelationship(currentUserId, targetUserId);
    
    return Result.success("绑定成功");
}
```

#### 切换流程
```
用户点击切换 → 验证会话 → 验证权限 → 创建新会话 → 更新活跃时间 → 返回新会话信息
```

#### 关键实现
```java
// 切换账户核心逻辑
public Result switchAccount(String token, String deviceId, Long targetUserId) {
    // 1. 获取分布式锁
    String lockValue = concurrentControlService.acquireLock(
        currentUserId.toString(), deviceId, "switch_account");
    
    try {
        // 2. 验证绑定关系
        if (!accountRelationshipService.isBound(currentUserId, targetUserId)) {
            return Result.error("无权限切换该账户");
        }
        
        // 3. 创建新会话
        UserSession newSession = createNewSession(targetUserId, deviceId, "switch");
        
        // 4. 更新缓存
        cacheSession(newSession);
        
        return Result.success(newSession);
        
    } finally {
        // 5. 释放锁
        concurrentControlService.releaseLock(
            currentUserId.toString(), deviceId, "switch_account", lockValue);
    }
}
```

```java
public String acquireLock(String userId, String deviceId, String operationType) {
    String lockKey = DISTRIBUTED_LOCK_PREFIX + userId + ":" + deviceId + ":" + operationType;
    String lockValue = UUID.randomUUID().toString();
    
    // Lua脚本实现原子获取锁
    String luaScript = "local key = KEYS[1] " +
            "local value = ARGV[1] " +
            "local expire = ARGV[2] " +
            "if redis.call('setnx', key, value) == 1 then " +
            "    redis.call('expire', key, expire) " +
            "    return 1 " +
            "else " +
            "    return 0 " +
            "end";
    
    Long result = redisTemplate.execute(script, Collections.singletonList(lockKey), 
            lockValue, LOCK_EXPIRE_TIME);
    
    return result != null && result == 1 ? lockValue : null;
}
```

```java
// 会话验证核心逻辑
private Long validateSession(String token, String deviceId) {
    // 1. 从Redis获取会话
    UserSession session = getSessionFromCache(token);
    
    // 2. 验证会话有效性
    if (session == null || session.getSessionStatus() != 1) {
        return null;
    }
    
    // 3. 验证设备绑定
    if (!session.getDeviceId().equals(deviceId)) {
        log.warn("设备不匹配 - 会话ID: {}, 期望设备: {}, 实际设备: {}", 
                token, session.getDeviceId(), deviceId);
        return null;
    }
    
    // 4. 验证过期时间
    if (session.getExpireTime().isBefore(LocalDateTime.now())) {
        invalidateSession(token);
        return null;
    }
    
    // 5. 更新最后活跃时间
    updateLastActiveTime(token);
    
    return session.getUserId();
}
```

## 📊 性能优化机制

### 1. 缓存策略

#### 多级缓存设计
```java
// 缓存穿透保护
public <T> T getOrLoadCache(String cacheKey, long expireSeconds, 
                           Supplier<T> databaseSupplier) {
    try {
        // 1. 尝试从Redis获取
        T value = (T) redisTemplate.opsForValue().get(cacheKey);
        if (value != null) {
            log.debug("缓存命中 - 缓存键: {}", cacheKey);
            return value;
        }
        
        // 2. 缓存未命中，从数据库获取
        T dbValue = databaseSupplier.get();
        if (dbValue != null) {
            // 3. 异步更新缓存（不阻塞当前请求）
            final T finalValue = dbValue;
            new Thread(() -> {
                try {
                    redisTemplate.opsForValue().set(cacheKey, finalValue, 
                            expireSeconds, TimeUnit.SECONDS);
                    log.debug("缓存更新成功 - 缓存键: {}", cacheKey);
                } catch (Exception e) {
                    log.warn("缓存更新失败 - 缓存键: {}, 错误: {}", cacheKey, e.getMessage());
                }
            }).start();
            return dbValue;
        }
        
        return value;
        
    } catch (Exception e) {
        // 4. Redis异常，降级到数据库
        log.warn("Redis异常，降级到数据库查询 - 缓存键: {}, 错误: {}", cacheKey, e.getMessage());
        return databaseSupplier.get();
    }
}
```

```javascript
            this.showError('切换失败: ' + error.message);
        }
    }
}
```
